一、中间按钮的架构哲学

1.1 定义与设计逻辑

中间按钮（Center Toggle）在Quantumult中承担着功能枢纽的角色，其设计遵循"最小化操作路径"原则：
- 空间位置：占据界面黄金视觉区域，确保拇指可轻松触达
- 状态反馈：通过色彩/图标变化实时显示代理开关、节点连接等状态
- 功能聚合：将高频操作（如代理切换、节点选择）从多级菜单提取到表层

1.2 技术实现原理

通过解析配置文件（.conf）中的[Policy]和[Server]模块，中间按钮实际上是一个可视化策略执行器。当用户点击时，会自动触发以下过程：

FAQ

PassWall2 的“节点切换策略”如何配置？
在 PassWall2 中可设置多节点策略，如“按延迟排序”、“自动选择最快节点”等。程序会定期测速，根据实时数据自动切换线路，保证连接的稳定性和速度。
Mellow 的 TUN 模式与 SOCKS5 代理有什么区别？
TUN 模式拦截系统 IP 层流量，实现全局代理；SOCKS5 代理通常只影响支持代理的应用。TUN 更适合全局加速和游戏、视频等应用，保证所有流量通过节点，提高网络稳定性。
Clash 如何配置策略组优先节点？
在 Clash 中，用户可以通过策略组设置节点优先级，优先使用延迟低、稳定性高的节点。结合规则文件，可以根据域名或 IP 自动选择最优节点，实现更高效的网络访问体验。
Clash 订阅被墙或无法下载如何绕过？
若订阅源被直接屏蔽，可尝试先在境外服务器上用代理代理获取订阅并通过中转服务器公开一个新的链接；或者使用 HTTPS 与 CDN 中转、或将订阅以文件形式托管在不易被封的云存储上。
Hiddify Next 用户流量配额如何设置？
管理员可为每个用户设置带宽和流量上限，超出限制时可降速或阻止访问。确保服务器资源合理分配，避免单用户占用过多带宽，提高整体网络公平性和使用效率。
V2RayN 的作用是什么？
V2RayN 是 V2Ray 的 Windows 图形客户端，提供节点管理、订阅导入、日志查看等功能。它支持多协议配置，并可与系统代理集成，是 Windows 用户最常用的代理管理工具之一。
Nekoray 的节点排序依据是什么？
Nekoray 默认按节点延迟排序。用户可以自定义排序规则，如按订阅来源、名称或连接成功率排列，从而快速找到性能最佳或最常用的节点。
VLESS + XTLS 模式在移动网络下的优势是什么？
XTLS 模式减少握手次数和加密开销，适合高延迟或不稳定的移动网络。结合策略组和多出口节点使用，可提高访问速度和连接稳定性，保证用户体验流畅可靠。
VMess 节点的 alterId 参数还需要设置吗？
在新版 V2Ray/Xray 内核中，alterId 已被废弃，不再影响认证逻辑。建议用户设置为 0 或删除该字段，以避免连接异常或不兼容情况。
SSR 的端口复用有什么风险？
SSR 支持多端口配置以复用服务，但若端口过多或设置冲突，可能被扫描工具识别或误判为异常服务。建议保持配置简洁，并合理分配端口范围以减少暴露风险。
